# Задание 1 — Витрина по взысканию

## Цель

Собрать простую аналитическую витрину для контроля результатов взыскания.

* Приведение дат к формату datetime

## Рассчитанные метрики

По каждому `dpd_bucket`:

* `total_cases` — общее число кейсов
* `contacted_cases_7d` — кейсы с контактом за 7 дней
* `paid_cases_7d` — кейсы с оплатой за 7 дней
* `collected_amount_7d` — сумма оплат
* `avg_payment_per_paid_case_7d` — средний платёж

# Задание 2 — Анализ A/B-теста

## Цель

Оценить влияние нового текста SMS на поведение клиентов.

## Подход

Рассчитаны метрики:

* `delivered_rate` — доля доставленных SMS
* `payment_rate_7d` — доля оплат за 7 дней
* `avg_paid_amount_per_customer_7d` — средний платёж

## Uplift

* Абсолютный uplift (основной)
* Относительный uplift (дополнительно)

## Проверка гипотез

* H0: различий между группами нет
* H1: различия есть

Использованы тесты:

* **z-test для сравнения долей (основной)**
* chi-square test (дополнительная проверка)

## Результат

* В группе `treatment` наблюдается рост `payment_rate_7d`
* uplift ≈ **+3.17 п.п.**
* p-value < 0.05 → различия статистически значимы

## Вывод

Новый текст SMS оказывает положительное и статистически значимое влияние на вероятность оплаты.

# Задание 3 — Построение витрины признаков

## Цель

Сформировать признаки для обучения модели на основе событийной истории.

## Важное условие

Используются только события:

```
action_dttm < snapshot_date
```

→ утечка данных (data leakage) исключена

---

## Построенные признаки

Для каждой пары `(customer_id, snapshot_date)`:

* `calls_last_7d` — звонки за 7 дней
* `sms_last_7d` — SMS за 7 дней
* `promises_last_30d` — обещания за 30 дней
* `payments_amount_last_30d` — сумма платежей за 30 дней
* `promised_amount_last_30d` — сумма обещаний
* `days_since_last_payment` — дней с последнего платежа

---

## Методология

* объединение snapshot и событий
* фильтрация по времени (исключение будущего)
* расчёт оконных агрегатов (7 и 30 дней)
* сборка итоговой витрины (`feature_mart`)

---

## Дополнительно

Найдены топ-20 клиентов:

* с максимальным числом контактов
* без оплат за последние 30 дней

---

## Особенности реализации

* Для count/sum-признаков отсутствие событий → 0
* Для `days_since_last_payment` отсутствие платежей → NaN
* Все временные окна строго до `snapshot_date`
