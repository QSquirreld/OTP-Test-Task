# Тестовое задание OTP Bank

## Общее описание

В репозитории представлены решения тестового задания, состоящего из трёх блоков:

- SQL-аналитика  
- Python-аналитика  
- Машинное обучение  

---

## Требования

### Среда
- Python 3.10+
- PostgreSQL

### Используемые библиотеки (Python)
- pandas  
- numpy  
- scipy  
- scikit-learn  
- xgboost / catboost / lightgbm  

---

## SQL-аналитика

### Описание

Анализ эффективности стратегий взыскания:  
контакты с клиентами, оплаты и прохождение по воронке.

### Задачи

#### Задача 1 — Эффективность стратегий (7 дней)
Коротко: сравнение стратегий по контактам и оплатам в первые 7 дней.  
Результат: наиболее сильные стратегии — `late_stage` и `mixed`, наиболее слабая — `sms_first`.

---

#### Задача 2 — Последний контакт и оплата
Коротко: определение последнего контакта по кейсу и проверка оплаты после него.  
Результат: получена таблица на уровне кейса с каналом контакта и фактом/суммой оплаты после него.

---

#### Задача 3 — Воронка обещаний
Коротко: анализ прохождения кейсов через этапы RPC → Promise → Payment.  
Результат: построена воронка по неделям и стратегиям с конверсиями и суммой сборов.

---

## Python-аналитика

### Описание

Решение задач по очистке данных, A/B-анализу и построению витрины признаков.

### Задачи

#### Задача 1 — Витрина по взысканию

Коротко: очистка данных и расчёт метрик взыскания по dpd-бакетам.
Результат: Получена витрина с метриками контактов и оплат за 7 дней по dpd-бакетам.

#### Задача 2 — Анализ A/B-теста SMS

Коротко: сравнение групп control и treatment и проверка статистической значимости.
Результат: Treatment показывает рост payment_rate (~+3.17 п.п.), различия статистически значимы (p-value < 0.05).

#### Задача 3 — Дневная витрина признаков

Коротко: построение признаков по событиям с учётом временных окон без утечки данных.
Результат: Собрана витрина признаков без утечки данных и выделены топ-20 клиентов с высокой активностью без оплат.

## Машинное обучение

### Описание

Построение модели вероятности оплаты, сравнение моделей и оценка их бизнес-эффекта.

### Задачи

#### Задача 1 — Baseline модель

Коротко: построение базовой модели для предсказания вероятности оплаты.
Результат: Получена baseline-модель с устойчивым качеством, пригодная для дальнейшего сравнения.

#### Задача 2 — Бустинг и сравнение моделей

Коротко: обучение модели бустинга и сравнение с baseline.
Результат: Бустинг не дал существенного улучшения качества по сравнению с baseline.

#### Задача 3 — Порог, бизнес-эффект и мониторинг

Коротко: выбор порога, расчёт экономического эффекта и анализ стабильности score.
Результат: Новая стратегия увеличивает экономический эффект и рекомендуется к запуску при контроле стабильности модели.
