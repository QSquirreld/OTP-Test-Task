# SQL-3: Воронка обещаний и оплат по стратегиям

## Задача
Сравнить стратегии взыскания не только по факту контакта, но и по глубине прохождения клиента через воронку:

`RPC → Promise → Kept Promise`

Дополнительно посчитать сумму сборов за 14 дней.

## Что считалось
Итоговая таблица построена по:
- `week_start`
- `strategy_name`

Рассчитаны поля:
- `total_cases`
- `rpc_cases_7d`
- `promise_cases_7d`
- `kept_promise_cases`
- `recovery_amount_14d`
- `rpc_rate_7d`
- `promise_rate_from_rpc`
- `kept_promise_rate`

## Подход
1. Сформирована база кейсов за период `2026-01-01` — `2026-03-31`
2. Для каждого кейса определена неделя старта:
   - `week_start = date_trunc('week', start_date)`
3. Отдельно посчитаны:
   - общее число кейсов
   - кейсы с `RPC` в первые 7 дней
   - кейсы с обещанием в первые 7 дней
   - сумма положительных платежей за 14 дней
4. Для `kept_promise_cases` отдельно найдено первое обещание по кейсу в окне 7 дней
5. После этого проверено, покрывает ли сумма положительных платежей по `account_id` до `promise_to_pay_date` включительно обещанную сумму

## Допущения
- Окно 7 дней реализовано как календарное:
  - `event_dttm::date BETWEEN start_date AND start_date + 6`
- Окно 14 дней реализовано как:
  - `payment_dttm::date BETWEEN start_date AND start_date + 13`
- Первое обещание определяется через `MIN(event_dttm)` с последующим join обратно к `contact_events`
- Предполагается, что для одного `case_id` нет нескольких promise-событий с одинаковым минимальным `event_dttm`, иначе возможны дубликаты строк

## Что даёт результат
Итоговая таблица позволяет:
- сравнивать стратегии по неделям старта кейсов
- видеть глубину прохождения воронки от контакта до обещания и выполнения обещания
- оценивать не только конверсии, но и денежный результат через `recovery_amount_14d`

## Как интерпретировать ключевые метрики
- `rpc_rate_7d = rpc_cases_7d / total_cases`  
  доля кейсов, по которым удалось дойти до `RPC`
- `promise_rate_from_rpc = promise_cases_7d / rpc_cases_7d`  
  доля кейсов с обещанием среди кейсов, где был `RPC`
- `kept_promise_rate = kept_promise_cases / promise_cases_7d`  
  доля выполненных обещаний среди кейсов с обещанием
- `recovery_amount_14d`  
  фактический денежный результат стратегии на горизонте 14 дней

## Вывод
Этот результат можно использовать как основу для сравнительного анализа стратегий по неделям: смотреть, какие стратегии лучше проводят кейс по воронке и какие дают лучший денежный результат на горизонте 14 дней.
